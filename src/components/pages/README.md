# pages

In Atomic Design, **pages** are concrete instances of templates filled with real,
representative content. They are inherently application-specific, so they do not
ship in this reusable component library.

The host app's pages live under `components/docs/` (for example `PracticalDemo`,
which fills the `Section` template with real research content). This folder is
kept as a deliberate placeholder to make the full atomic hierarchy explicit.
